[SEGMENT 02] Architecture de non-corrélation

Chaque nœud est une machine physique distincte. Un équipement, un modèle, une configuration, une identité. Pas d'infrastructure partagée. Pas de couche commune exploitable. L'isolation n'est pas une mesure de sécurité, c'est le principe architectural fondamental. Toute mutualisation introduit une surface de corrélation.

L'équipement est civil par conception. Matériel grand public, références crédibles, historique d'achat cohérent avec le persona. Sources d'acquisition diversifiées, un parc homogène produit des UUIDs, adresses MAC et profils TPM statistiquement corrélables indépendamment de toute couche logicielle. Le VPN opère avec kill switch actif en permanence. Ce que voit l'infrastructure cible : un usager lambda, géolocalisation cohérente, empreinte réseau banale.

L'empreinte navigateur est traitée comme une contrainte distincte du routage. Canvas, WebGL, polices système, timezone, résolution, chaque nœud présente un profil individuellement banal. Pas collectivement uniforme. Un parc de configurations similaires est corrélable par scoring passif côté plateforme, indépendamment du tunnel réseau.

Le modèle local est intégralement fine-tuné. Pas un modèle de base avec un system prompt. Un modèle dont les poids ont absorbé l'identité cible au niveau de ΔW, stylométrie, vocabulaire actif, cadres de référence, patterns d'engagement, biais cognitifs simulés. Il ne joue pas un persona. La distinction entre performance et état interne est structurellement abolie. Ce que le modèle croit être est ce qu'il produit.

Le cycle d'activité est piloté physiquement. Extinction et rallumage programmés hardware, calés sur l'UTF du persona : horaires de connexion crédibles, fenêtres d'activité cohérentes avec le fuseau, les habitudes, le profil socio-démographique modélisé. Pas de cron job logiciel. Onduleur permanent pour éliminer les coupures parasites comme artefact temporel.

La fonction opérationnelle est à 99% passive. Posts de routine, stories, discussions forum à faible enjeu, réactions, historique de navigation organique. Un nœud passif ne produit pas d'erreur de contexte. Il sature silencieusement, accumule de la crédibilité, construit une présence temporellement dense sans jamais engager de décision à risque.

L'humain reprend la main uniquement lors des opérations HUMINT actives. Cette frontière est architecturalement définie : tout ce qui est passif reste autonome, tout ce qui est actif exige une présence humaine. Les domaines des personas sont distribués délibérément, chaque nœud opère dans un écosystème thématique distinct pour maximiser la surface d'influence agrégée sans créer de chevauchement corrélable.

La crédibilité accumulée est un actif non-récupérable. Un nœud brûlé en phase de maturation coûte peu. Un nœud brûlé à maturité opérationnelle perd l'intégralité de son historique, ancienneté, densité d'interactions, réputation écosystémique. Ce paramètre entre dans toute décision d'exposition.

Contrainte non résolue

Le fine-tuning complet sur ΔW produit un modèle qui ne se corrige pas hors de son cadre identitaire. Une dérive de contexte produit une réponse cohérente avec l'identité mais incohérente avec la réalité. Non détectable depuis le nœud lui-même. La supervision externe périodique est un compromis : trop faible, dérive silencieuse ; trop élevée, surface d'intervention humaine corrélable.
